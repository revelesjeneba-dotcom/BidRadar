## 当前风险

- `main.py` 默认 `USE_SAMPLE_DATA = False`，运行前仍需明确联网和正式数据写入授权。
- 外部网站结构、访问限制和登录策略可能变化，离线回归不能替代受控的采集验收。
- Excel 被桌面应用占用或目标目录无写权限时，安全写入会明确失败；应使用自动备份恢复而不是强制覆盖。
- 多文件流程采用备份与补偿恢复，不是数据库级事务；异常后需核对错误信息列出的备份路径。

## 后续建议

- V6.6.0 稳定期只处理缺陷，不混入业务规则调整。
- 如需真实采集验收，单独制定授权、限频、隔离输出和回滚方案。
- 后续版本再评估依赖锁定、日志结构化和多文件事务增强。
