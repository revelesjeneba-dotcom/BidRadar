# Development Rules

1. 禁止修改业务逻辑时顺便重构。
2. 一次 Commit 只做一类修改。
3. 新增路径必须登记 `paths.py`。
4. 新增 Excel 必须登记 `FILE_MAP.md`。
5. 新增模块必须更新 `docs/MODULE_INDEX.md`。
6. 任何重构必须保持业务结果一致。

## 路径规则

- 业务脚本不得新增硬编码业务文件名。
- 核心业务路径从 `paths.py` 获取。
- 改目录时优先修改 `paths.py`，不要分散修改业务脚本。

## Excel 规则

- 新增 Excel 输出必须说明生成脚本、数据类型和覆盖风险。
- 写正式业务 Excel 前应确认是否需要备份。
- 不要在调试脚本中写正式结果文件。

## 验证规则

- Windows 环境优先使用 `python -B` 避免 `__pycache__` 权限问题。
- 真实采集和模拟测试必须分开记录。
- 重构前后应对关键统计保持一致。
